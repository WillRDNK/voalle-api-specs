# APIs Existentes com Lacunas

Este diretório reúne APIs já existentes, mas que hoje apresentam algum tipo de limitação para uso prático na operação ou nas integrações.

Aqui devem ser documentados casos como:

- APIs que existem, mas não atendem totalmente a necessidade de negócio;
- rotas com comportamento inconsistente;
- endpoints com retorno insuficiente;
- funcionalidades incompletas;
- limitações de autenticação, paginação, filtros, payload ou regras de negócio;
- APIs que exigem ajustes para se tornarem viáveis em automações.

## Objetivo

Criar um inventário claro de lacunas nas APIs atuais para facilitar:

- priorização de melhorias;
- discussão com times internos e com a Voalle;
- registro do impacto operacional e técnico;
- definição do comportamento esperado.

## Como usar

Para cada API com problema ou incompletude:

1. copie o arquivo `template.md`;
2. renomeie com um nome descritivo;
3. preencha com o problema atual, impacto e necessidade esperada.
